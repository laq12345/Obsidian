# sd vs sed — 查找替换文本完全指南

> **sd**（Rust 写，现代） vs **sed**（GNU，经典标准）
> 两者都是流式文本编辑器，但设计哲学截然不同。

---

## 1. 一句话区别

| 工具 | 哲学 | 适合 |
|------|------|------|
| **sd** | "我就做个查找替换" | 日常查找替换、正则替换、批量文件修改 |
| **sed** | "我是流编辑器" | 行操作（删除/插入/打印）、复杂脚本、跨平台兼容 |

---

## 2. 安装

```bash
# sd — 系统已预装（Fedora 默认）
cargo install sd

# sed — 系统已预装（GNU sed 4.9）
# Fedora 默认已带，无需额外安装
```

## 3. 语法对比

### 3.1 基本替换

```bash
# sd：参数直觉，<查找> <替换> [文件...]
sd 'old' 'new' file.txt

# sed：需要 s 命令和分隔符
sed 's/old/new/' file.txt       # 只替换每行第一个
sed 's/old/new/g' file.txt      # 全局替换
```

### 3.2 原地修改文件

```bash
sd 'old' 'new' file.txt                  # 默认就是原地修改
sd -p 'old' 'new' file.txt               # 预览模式（不实际写入）

sed -i 's/old/new/g' file.txt            # -i 才能原地修改
sed -i.bak 's/old/new/g' file.txt        # 备份原文件为 .bak
```

### 3.3 从 stdin 读取

```bash
echo "hello world" | sd 'hello' 'hi'     # 直接管道
echo "hello world" | sed 's/hello/hi/'   # 也一样
```

## 4. 核心功能详细对比

### 4.1 查找替换

| 场景 | sd | sed |
|------|----|-----|
| 第一个匹配 | `sd old new` | `sed 's/old/new/'` |
| 全部匹配 | `sd old new`（默认就是全部） | `sed 's/old/new/g'` |
| 第 N 个匹配 | ❌ 不支持 | `sed 's/old/new/2'` |
| 第 N 个及之后 | ❌ 不支持 | `sed 's/old/new/2g'` |
| 忽略大小写 | `sd -f i old new` | `sed 's/old/new/I'`（GNU 扩展） |
| 限制替换次数 | `sd -n 5 old new` | `sed 's/old/new/5'`（只到第5个） |
| 预览不写入 | `sd -p old new file` | 无原生预览（用 `sed 's/old/new/g' file \| diff file -`） |

### 4.2 正则能力

| 特性 | sd | sed |
|------|----|-----|
| 正则方言 | JavaScript/Python 风格（Rust regex crate） | 基本 BRE（默认）/ ERE（`-r` 或 `-E`） |
| 捕获组 | `$1`, `$2`, `$name` | `\1`, `\2` |
| 命名捕获组 | `(?P<name>...)` + `$name` | ❌ 不支持 |
| 跨行匹配 | `-A` 选项 | 需要 `N`/`P` 等命令组合 |
| 非贪婪匹配 | 默认就是非贪婪 | 需要 `[^...]*` 手动模拟 |
| `.` 匹配换行 | `-f s` | ❌ 不支持 |
| 全文匹配 | `-f m` | `:a;N;$!ba;...` | 
| 转义 `$` | `$$` | `\$` |

```bash
# sd：命名捕获组
echo "123.45" | sd '(?P<dollars>\d+)\.(?P<cents>\d+)' '$dollars dollars and $cents cents'
# 输出: 123 dollars and 45 cents

# sed：只能用索引捕获
echo "123.45" | sed -r 's/([0-9]+)\.([0-9]+)/\1 dollars and \2 cents/'
# 输出: 123 dollars and 45 cents
```

### 4.3 行操作

| 操作 | sd | sed |
|------|----|-----|
| 删除第 N 行 | ❌ 不支持 | `sed '5d'` |
| 删除匹配行 | ❌ 不支持 | `sed '/pattern/d'` |
| 打印第 N 行 | ❌ 不支持 | `sed -n '5p'` |
| 删除空行 | `sd '^\s*$' ''`（保留空行但清空） | `sed '/^$/d'`（真正删除行） |
| 行前插入 | ❌ 不支持 | `sed '5i\...'` |
| 行后追加 | ❌ 不支持 | `sed '5a\...'` |
| 替换整行 | ❌ 不支持 | `sed '/pattern/c\...'` |
| 行号输出 | ❌ 不支持 | `sed '='` |

**结论：** 如果你需要行级别的操作（删除行、插入行、打印指定行），**必须用 sed**。sd 只做查找替换。

### 4.4 多命令与脚本

```bash
# sd：不支持多条命令组合，一次只做一个替换

# sed：可以组合多条命令
sed -e 's/foo/bar/' -e 's/baz/qux/' file.txt
sed 's/foo/bar/; s/baz/qux/' file.txt
sed -f script.sed file.txt                # 从脚本文件读取
```

## 5. 常见任务对照表

| 任务 | sd | sed |
|------|----|-----|
| 替换所有 old 为 new | `sd old new file` | `sed -i 's/old/new/g' file` |
| 只替换第一个 | ❌（sd 默认全部） | `sed 's/old/new/'` |
| 忽略大小写替换 | `sd -f i old new file` | `sed -i 's/old/new/I' file` |
| 字面量替换（禁正则） | `sd -F '$HOME' '/home/user'` | 需要疯狂转义 `sed 's/\$HOME/\/home\/user/g'` |
| 去除行尾空白 | `sd '\s+$' '' file` | `sed -i 's/[[:space:]]\+$//' file` |
| 删除空行 | ❌ 不能真正删除行 | `sed -i '/^$/d' file` |
| 删除注释行 | ❌ 不能删除行 | `sed -i '/^#/d' file` |
| 打印第5-10行 | ❌ | `sed -n '5,10p' file` |
| 提取 IP | ❌ | `sed -n 's/.*addr:\([0-9.]*\).*/\1/p'` |
| 替换路径中的斜杠 | `sd '/usr' '/opt'`（无歧义） | `sed 's#/usr#/opt#g'`（需换分隔符） |
| 跨行替换 `\r\n` → `\n` | `sd -A '\r\n' '\n' file` | `sed -i 's/\r$//' file` |
| 批量替换（配合 fd） | `fd -e py -x sd old new` | `fd -e py -x sed -i 's/old/new/g'` |
| 修改前备份 | ❌ | `sed -i.bak 's/old/new/g' file` |
| 在文件头尾加内容 | ❌ | `sed -i '1i\HEADER' ; ' $a\FOOTER' file` |

## 6. 性能对比

```
sd v1.0.0 vs GNU sed 4.9

1.5GB JSON 简单替换:  sd 快约 2.35 倍
55MB  JSON 正则替换:   sd 快约 11.93 倍
```

sd 用 Rust 实现，内存安全、多线程感知。sed 是 C 写的单线程流处理。

但在日常小文件上两者差异可忽略，选顺手而非选快的。

## 7. 实战建议

### 什么时候用 sd ✅

```bash
# 日常查找替换 — 语法最干净
sd 'window.fetch' 'fetch' src/*.js

# 字面量替换（含特殊字符）
sd -F '((([\]])))' '' file.txt

# 批量重命名/替换变量
fd -e ts -x sd 'oldFunction' 'newFunction'

# 带命名捕获组的复杂替换
sd '(?P<user>\w+)@(?P<domain>\w+\.\w+)' 'user: $user, domain: $domain' emails.txt

# 预览确认后再改
sd -p 'old' 'new' config.json
```

### 什么时候用 sed ✅

```bash
# 删除行 — sd 做不到
sed -i '/^#/d; /^$/d' config.conf     # 删注释行+空行
sed -i '3,8d' file.txt                  # 删第3-8行

# 提取特定行范围
sed -n '20,50p' huge.log > subset.log

# 在文件头尾插入内容
sed -i '1i\# Generated by pipeline' output.txt
sed -i '$a\EOF' output.txt

# 行级别的复杂脚本
sed -n '/ERROR/,/END_ERROR/p' logfile   # 打印两个模式之间的行

# 跨平台脚本（AIX/Solaris 也有 sed）
# sd 只 Linux/macOS
```

### 组合使用

```bash
# sed 删行，sd 替换 — 各取所长
sed -i '/^#/d' config.ini
sd 'old' 'new' config.ini
```

## 8. sd 选项速查

```
sd [OPTIONS] <FIND> <REPLACE_WITH> [FILES]...

  -p, --preview              预览，不实际写入
  -F, --fixed-strings        字面量模式（禁正则）
  -n, --max-replacements N   限制每文件替换次数（0=不限制）
  -f, --flags <FLAGS>        regex 标志，可组合：
       c  — 大小写敏感（默认）
       e  — 禁用多行模式
       i  — 忽略大小写
       m  — 多行匹配
       s  — 让 . 匹配换行符
       w  — 仅匹配完整单词
```

## 9. sed 选项速查

```
sed [选项] '命令' [文件...]

  -i[SUFFIX]   原地修改文件（可带备份后缀，如 -i.bak）
  -e           添加命令
  -n           静默模式（只打印处理过的行）
  -r / -E      使用扩展正则（ERE）
  -f           从文件读取 sed 脚本

核心命令：
  s/old/new/   替换（加 g 全局、数字限制次数）
  p            打印行（配合 -n）
  d            删除行
  a/i          行后追加 / 行前插入
  c            替换整行
  y            字符转换
  q            退出
  =            打印行号
  r/w          读取/写入文件
  N/P/D        多行模式命令
```

## 10. 转义对照表

| 特殊字符 | sd（替换侧） | sed（替换侧） |
|---------|-------------|--------------|
| `$` | `$$` | `\$` |
| `\` | `\\` | `\\` |
| `&`（原样） | 无需转义 | `\&` |
| `/` | 无需转义 | 可选其他分隔符（如 `#`） |
| 换行符 | `\n` | `\n`（某些版本） |
| 捕获组引用 | `$1`, `$name` | `\1`, `\&` |

## 11. 一句话选型指南

```
只想替换文本？              → sd
要删除行/插入行/打印行？    → sed
写跨平台 shell 脚本？       → sed（POSIX 标准）
日常批量替换代码？          → sd
做复杂文本过滤？            → sed + awk
```
