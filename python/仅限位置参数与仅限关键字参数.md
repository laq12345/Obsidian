在 Python 函数定义中，`/` 和 `*` 是两个特殊的符号，用于精确控制参数的传递方式，它们被称为“形参分隔符”。

### 🔢 `/` (斜杠)：仅限位置参数

`/` 符号用于分隔参数。它**左侧**的所有参数都必须是**位置参数**，不能通过关键字参数的形式传递。

- **目的**：强制调用者按顺序传参，通常用于核心、不可或缺的数据，或者为了 API 的向后兼容性（允许未来修改参数名而不破坏现有代码）。

**示例：**

```python
def divide(a, b, /, c):
    """
    a 和 b 是仅限位置参数，c 可以是位置或关键字参数。
    """
    return (a / b) + c

# 正确用法
divide(10, 2, 5)        # a=10, b=2, c=5
divide(10, 2, c=5)      # a=10, b=2, c=5

# 错误用法：a 和 b 不能使用关键字传参
# divide(a=10, b=2, c=5) # 会抛出 TypeError
# divide(10, b=2, c=5)   # 会抛出 TypeError
```

### 🔑 `*` (星号)：仅限关键字参数

`*` 符号同样用于分隔参数。它**右侧**的所有参数都必须是**关键字参数**，不能通过位置参数的形式传递。

- **目的**：提高代码可读性，强制调用者为某些参数显式命名，避免混淆。这在函数有多个可选配置项时非常有用。

**示例：**

```python
def create_user(name, *, age, city="Unknown"):
    """
    name 可以是位置或关键字参数，但 age 和 city 必须是关键字参数。
    """
    return {"name": name, "age": age, "city": city}

# 正确用法
create_user("Alice", age=30)                # name="Alice", age=30, city="Unknown"
create_user(name="Bob", age=25, city="NYC") # name="Bob", age=25, city="NYC"

# 错误用法：age 和 city 不能按位置传递
# create_user("Charlie", 30, "LA") # 会抛出 TypeError
```

### 🧩 组合使用 `/` 和 `*`

你可以同时使用这两个符号，来创建一个参数规则非常明确的函数。`/` 和 `*` 之间的参数则遵循默认行为，即既可以通过位置也可以通过关键字传递。

一个经典的例子是内置的 `sorted()` 函数，它的签名类似于： `sorted(iterable, /, *, key=None, reverse=False)`

我们可以自定义一个类似的函数：

```python
def my_function(pos_only, /, standard, *, kw_only):
    """
    pos_only: 仅限位置参数
    standard: 位置或关键字参数均可
    kw_only:  仅限关键字参数
    """
    print(f"{pos_only=}, {standard=}, {kw_only=}")

# 正确用法
my_function(1, 2, kw_only=3)          # pos_only=1, standard=2, kw_only=3
my_function(1, standard=2, kw_only=3) # pos_only=1, standard=2, kw_only=3

# 错误用法
# my_function(pos_only=1, standard=2, kw_only=3) # pos_only 不能是关键字参数
# my_function(1, 2, 3)                           # kw_only 不能是位置参数
```

### ✨ 与 `*args` 和 `**kwargs` 的区别

请注意，这里的 `*` 是作为分隔符，与用于收集可变数量参数的 `*args` 和 `**kwargs` 是不同的概念。

- **`*args`**: 收集任意数量的**位置参数**到一个元组中。
- **`**kwargs`**: 收集任意数量的**关键字参数**到一个字典中。
- **`*` (分隔符)**: 仅用于标记其后的参数必须是关键字参数，它本身不收集任何参数。
